# BibleRogue2 채팅 인수인계 프로토콜

이 문서는 새 채팅에게 작업을 맡길 때 반드시 전달해야 하는 기준 문서이다. 기존 대화 메모리가 없어도 이 문서와 관련 공식 문서를 기준으로 작업을 이어갈 수 있어야 한다.

---

## 1. 기본 원칙

모든 작업 채팅은 다음 원칙을 따른다.

1. 맡은 역할 외의 파일을 수정하지 않는다.
2. 수정 금지 파일을 임의로 건드리지 않는다.
3. 작업 전 현재 main 기준 파일 상태를 확인한다.
4. 보고서, ZIP 목록, 기억이 아니라 GitHub main의 실제 파일을 기준으로 판단한다.
5. validate 또는 브라우저 QA를 직접 실행하지 못했다면 완료라고 말하지 않는다.
6. 다른 채팅의 보고는 참고 자료일 뿐, 최종 근거는 실제 파일과 실행 결과이다.
7. 패치 범위가 넓어질 것 같으면 총괄에게 보고하고 멈춘다.

---

## 2. 총괄 챗 역할

총괄 챗은 다음을 담당한다.

- 현재 프로젝트 상태 판단
- 작업 우선순위 결정
- 각 채팅 역할 분배
- 수정 가능/금지 파일 지정
- 다른 채팅 보고서 검토
- 실제 main 기준 반영 여부 확인
- 최종 QA 기준 수립
- 새 챕터 작업 시작 승인

총괄 챗은 직접 대규모 코드를 바꾸기보다, 원칙적으로 작업 지시와 검수, 위험 판단을 우선한다. 단, 긴급 오류 수정이 필요하고 범위가 명확할 때는 직접 패치할 수 있다.

---

## 3. 스토리 구현 챗 역할

스토리 구현 챗은 story data만 담당한다.

수정 가능 예시:

- `src/data/*StructurePatch.js`
- `src/data/endings.js`
- `docs/story/*.md`

수정 금지 예시:

- `index.html`
- `src/main.js`
- `src/chapter-runtime.js`
- `src/responsive.js`
- CSS 파일
- assets 파일

역할:

1. 노드 작성.
2. 선택지 작성.
3. effects 부여.
4. 즉시 배드엔딩 연결.
5. 최종 엔딩 연결.
6. story route checklist 작성.
7. image mapping 문서 작성.

주의:

- 플레이어는 성경 주인공이 아니다.
- 성경 사건 자체를 바꾸면 안 된다.
- `effects` key는 현재 런타임과 validator가 지원하는 값만 사용한다.
- 새 ending ID를 만들면 반드시 `endings.js`에 추가한다.
- `next` target 오타를 피한다.

---

## 4. 이미지/에셋 챗 역할

이미지/에셋 챗은 PNG 에셋 제작과 패키징만 담당한다.

수정 가능:

- `assets/images/**`
- `assets/ui/**` 중 명시된 경로
- 필요 시 이미지 보고용 txt 파일

수정 금지:

- 코드 파일
- CSS 파일
- story data 파일
- index.html

역할:

1. 지정된 파일명으로 PNG 제작.
2. 지정된 경로 구조 유지.
3. ZIP 패키징.
4. 실제 업로드 후 파일명 목록 보고.
5. 누락 파일, 파일명 불일치, 경로 불일치 보고.

주의:

- 보고서 기준이 아니라 GitHub main 실제 폴더 기준으로 검수한다.
- 파일명 대소문자, 언더스코어, 하이픈을 정확히 맞춘다.
- 플레이 이미지는 보통 `play_left_520x650` 경로를 따른다.
- 엔딩 이미지는 보통 `original_16x9` 경로를 따른다.
- UI 프레임, 텍스트, 로고를 이미지에 넣지 않는다.

---

## 5. UI/런타임 챗 역할

UI/런타임 챗은 화면 구조, 라우팅, 런타임 연결만 담당한다.

수정 가능은 작업마다 총괄이 지정한다.

일반적으로 가능한 파일:

- `index.html`
- `src/main.js`
- `src/chapter-runtime.js`
- 특정 CSS 파일
- `tools/validate-story.js`

일반적으로 수정 금지:

- story data
- assets/images
- assets/ui/play 이미지
- 검증 없이 `responsive.js`에 스토리 로직 추가

주의:

- `responsive.js`는 반응형 화면 처리만 담당해야 한다.
- 챕터 라우팅과 이미지 경로 보정은 `chapter-runtime.js`에 둔다.
- `main.js`를 크게 갈아엎는 패치는 위험하다. 필요한 경우 총괄 승인 후 진행한다.
- 새 `setInterval`, `MutationObserver`, 전역 감시 루프는 금지한다.
- 흰 화면, RAM 폭주, 무한 렌더링 가능성이 있는 방식은 쓰지 않는다.

---

## 6. QA 챗 역할

QA 챗은 수정하지 않고 검수한다.

검수 항목:

1. `node tools/validate-story.js`
2. 홈 화면 로드
3. Part 전환
4. 챕터 진입
5. 선택지 진행
6. 즉시 배드엔딩 진입
7. 최종 엔딩 진입
8. 이미지 표시
9. 콘솔 오류
10. 404 오류
11. 모바일/가로 화면 주요 UI 깨짐

QA 챗은 문제 발견 시 다음 형식으로 보고한다.

```text
문제 위치:
재현 경로:
기대 결과:
실제 결과:
콘솔 오류:
의심 원인:
수정 필요 파일 후보:
```

QA 챗은 임의 수정하지 않는다.

## 7. 새 채팅에게 보낼 기본 상태 공유문

새 채팅을 만들 때 아래 문장을 함께 보낸다.

```text
BibleRogue2 공식 문서 기준으로 작업한다.
먼저 docs/project/OFFICIAL_PROJECT_STATE.md와 docs/project/CHAT_HANDOFF_PROTOCOL.md를 기준으로 현재 상태를 이해한다.
작업 전 반드시 수정 가능 파일과 수정 금지 파일을 구분한다.
보고서나 기억이 아니라 GitHub main의 실제 파일을 기준으로 판단한다.
```

---

## 8. 현재 권장 작업 분배

현재 단계 이후 권장 분배는 다음과 같다.

- 총괄 챗: 1장/2장 최종 QA 판정 및 3장 여리고 착수 승인.
- QA 챗: 출애굽/광야 전체 최종 점검.
- 스토리 챗 A: Part I. 3장 여리고 story data 설계 및 구현 준비.
- 이미지 챗: 여리고 홈/플레이/엔딩 이미지 제작 준비.
- UI/런타임 챗: 새 챕터 등록 시 `chapter-runtime.js` 확장만 수행.
